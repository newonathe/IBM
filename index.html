Ch 1: Big O Notation
How to measure how fast (or slow) your code is. Essential for writing solutions that don't time out.
⏱ 45 minutes

🤔 Why Does This Matter?

HackerRank problems have a time limit (usually 1–3 seconds). If your algorithm is too slow, you get "Time Limit Exceeded" (TLE) — which means 0 points even if your logic is correct. Big O helps you predict if your solution is fast enough before you even run it.

Big O describes how running time grows as input size n grows. We ignore constants and focus on the dominant term.

Recognizing Complexity in Code PYTHON
python3 — complexity examples
# ─── O(1): No loop, direct access ───────────────────────
def get_first(lst):
    return lst[0]          # Always 1 step regardless of len(lst)

# ─── O(n): One loop through n items ─────────────────────
def find_max(lst):
    best = lst[0]
    for x in lst:          # Loops n times
        if x > best:
            best = x
    return best

# ─── O(n²): Nested loops ────────────────────────────────
def has_duplicate(lst):
    for i in range(len(lst)):
        for j in range(i + 1, len(lst)):   # Inner loop: n-1 times
            if lst[i] == lst[j]:
                return True
    return False
# ⚠️ If n=10,000 this is 50 million comparisons!

# ─── O(n²) IMPROVED to O(n) using a set ─────────────────
def has_duplicate_fast(lst):
    seen = set()
    for x in lst:          # Only ONE loop now
        if x in seen:
            return True
        seen.add(x)        # set lookup/add = O(1)
    return False

# ─── O(log n): Halving the problem each step ────────────
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2    # Eliminate half each time
        if arr[mid] == target: return mid
        elif arr[mid] < target: lo = mid + 1
        else: hi = mid - 1
    return -1

# ─── O(n log n): Sorting (Python's sort is Timsort) ─────
lst = [3, 1, 4, 1, 5]
lst.sort()     # O(n log n) — Python's built-in sort

📏 Space Complexity

Besides time, Big O also measures memory usage. Space complexity matters when creating extra data structures.

  • O(1) space: You use a fixed number of variables — a few integers, etc.
  • O(n) space: You create a new list/dict of size n (e.g., frequency map)
  • O(n²) space: You create a 2D grid of size n×n (e.g., DP table)

On HackerRank, if you're within memory limits, usually O(n) space is fine.

Ch 2: Recursion
A function that calls itself. The most elegant technique in programming — and the most common source of confusion for beginners.
⏱ 60 minutes

🪆 The Core Idea

Recursion solves a big problem by breaking it into a smaller version of the same problem, over and over, until you hit a case small enough to solve directly.

Every recursive function has exactly two parts:

  • Base Case: The stopping condition. Without this, you get infinite recursion (stack overflow).
  • Recursive Case: The function calls itself with a smaller/simpler input.
Anatomy of Recursion PYTHON
python3 — recursive countdown
def countdown(n):
    # ── BASE CASE: stop condition ──
    if n <= 0:
        print("Done!")
        return        # ← MUST return here, otherwise recursion continues
    
    # ── RECURSIVE CASE ──
    print(n)
    countdown(n - 1)  # ← calls itself with SMALLER input

countdown(3)
# Output: 3, 2, 1, Done!

# ─── TRACING THE CALL STACK ───────────────────────────────
# countdown(3)
#   prints 3
#   calls countdown(2)
#     prints 2
#     calls countdown(1)
#       prints 1
#       calls countdown(0)
#         prints "Done!"
#         returns
#       returns
#     returns
#   returns
Classic Examples PYTHON
python3 — factorial, fibonacci, sum
# ═══ FACTORIAL ═══════════════════════════════════════════
# n! = n * (n-1)!    e.g. 5! = 5 * 4 * 3 * 2 * 1 = 120
# Base case: 0! = 1

def factorial(n):
    if n == 0:          # BASE CASE
        return 1
    return n * factorial(n - 1)   # RECURSIVE CASE

print(factorial(5))    # 120
# Trace: fact(5) = 5 * fact(4)
#                    = 5 * 4 * fact(3)
#                    = 5 * 4 * 3 * fact(2)
#                    = 5 * 4 * 3 * 2 * fact(1)
#                    = 5 * 4 * 3 * 2 * 1 * fact(0)
#                    = 5 * 4 * 3 * 2 * 1 * 1 = 120

# ═══ FIBONACCI ═══════════════════════════════════════════
# fib(n) = fib(n-1) + fib(n-2)
# Base cases: fib(0)=0, fib(1)=1

def fib(n):
    if n <= 0: return 0    # BASE CASE 1
    if n == 1: return 1    # BASE CASE 2
    return fib(n-1) + fib(n-2)   # RECURSIVE CASE

print(fib(7))    # 13 (0,1,1,2,3,5,8,13)
# ⚠️ WARNING: This is O(2^n) — EXPONENTIAL! Very slow.
# fib(40) takes seconds. Fix with memoization (Ch 10).

# ═══ SUM OF LIST ═════════════════════════════════════════
def sum_list(lst):
    if len(lst) == 0:       # BASE CASE: empty list = 0
        return 0
    return lst[0] + sum_list(lst[1:])   # head + sum of tail

print(sum_list([1, 2, 3, 4, 5]))   # 15

# ═══ STRING REVERSAL ═════════════════════════════════════
def reverse_str(s):
    if len(s) <= 1:         # BASE CASE
        return s
    return reverse_str(s[1:]) + s[0]  # reverse(tail) + first char

print(reverse_str("hello"))   # "olleh"

# ═══ POWER FUNCTION ══════════════════════════════════════
# x^n = x * x^(n-1)    BASE: x^0 = 1
def power(x, n):
    if n == 0: return 1
    return x * power(x, n - 1)

print(power(2, 10))    # 1024
Binary Search — Recursive Version PYTHON
python3
def binary_search_rec(arr, target, lo, hi):
    # BASE CASE: not found
    if lo > hi:
        return -1
    
    mid = (lo + hi) // 2
    
    if arr[mid] == target:
        return mid          # BASE CASE: found!
    elif arr[mid] < target:
        return binary_search_rec(arr, target, mid + 1, hi)  # search right half
    else:
        return binary_search_rec(arr, target, lo, mid - 1)  # search left half

arr = [1, 3, 5, 7, 9, 11, 13]
print(binary_search_rec(arr, 7, 0, len(arr)-1))   # 3 (index)
⚠️
The #1 Recursion Mistake Forgetting the base case, or having a base case that's never reached. Python's recursion limit is ~1000 calls deep. If your recursion goes deeper (e.g., for n > 1000), you'll get a RecursionError. Use iteration or memoization instead.

python3 — stack & queue implementation
from collections import deque

# ═══ STACK: use a Python list ════════════════════════════
# push = append(), pop = pop()

stack = []
stack.append(10)    # push 10
stack.append(20)    # push 20
stack.append(30)    # push 30

print(stack[-1])    # peek: 30 (top element without removing)
print(stack.pop())  # pop: 30 ← last in, first out!
print(stack.pop())  # pop: 20
print(stack)        # [10]

# ═══ QUEUE: use collections.deque ════════════════════════
# enqueue = append(), dequeue = popleft()
# ⚠️ Don't use a list for queue! list.pop(0) is O(n)
# deque.popleft() is O(1)

queue = deque()
queue.append("Alice")     # enqueue
queue.append("Bob")       # enqueue
queue.append("Charlie")   # enqueue

print(queue[0])           # peek front: Alice
print(queue.popleft())    # dequeue: Alice ← first in, first out!
print(queue.popleft())    # dequeue: Bob
print(queue)              # deque(['Charlie'])

# ═══ CLASSIC STACK PROBLEM: Balanced Parentheses ═════════
def is_balanced(s):
    """Check if brackets are properly balanced. e.g. '(())' → True, '([)]' → False"""
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    
    for char in s:
        if char in '([{':
            stack.append(char)           # push opening bracket
        elif char in ')]}':
            if not stack or stack[-1] != pairs[char]:
                return False             # no matching opener
            stack.pop()                  # pop the matched opener
    
    return len(stack) == 0               # True if nothing left

print(is_balanced("(())"))      # True
print(is_balanced("({[]})"))    # True
print(is_balanced("([)]"))      # False
print(is_balanced("((()"))      # False (unclosed)
MediumNext Greater Element (Stack)▶

For each element, find the next element to the right that is greater. If none, output -1.

Input
[2, 1, 2, 4, 3, 1, 5]
Output
[4, 2, 4, 5, 5, 5, -1]
python3 — monotonic stack
def next_greater(arr):
    n = len(arr)
    result = [-1] * n
    stack = []   # stores indices
    
    for i in range(n):
        # Pop elements that are smaller than current
        while stack and arr[stack[-1]] < arr[i]:
            idx = stack.pop()
            result[idx] = arr[i]   # current element is the answer
        stack.append(i)
    
    return result

arr = list(map(int, input().split()))
print(*next_greater(arr))

def find_duplicates(lst):
    seen = set()
    dups = set()
    for x in lst:
        if x in seen:
            dups.add(x)
        seen.add(x)
    return dups

def bubble_sort(arr):
    """
    Bubble sort: compare adjacent pairs, swap if out of order.
    Largest 'bubbles up' to end each pass.
    Time: O(n²) | Space: O(1)
    """
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n - i - 1):    # shrink range each pass
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]  # Python swap!
                swapped = True
        if not swapped:   # early exit if already sorted
            break
    return arr

# [11, 12, 22, 25, 34, 64, 90]

python3 — divide and conquer
def merge_sort(arr):
    """
    Merge sort: divide array in half, recursively sort each half, merge.
    Time: O(n log n) | Space: O(n)
    Stable sort — preserves relative order of equal elements.
    """
    if len(arr) <= 1:        # BASE CASE: single element = sorted
        return arr
    
    # DIVIDE: split in half
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])   # recursive sort left
    right = merge_sort(arr[mid:])  # recursive sort right
    
    # MERGE: combine two sorted halves
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    # Append remaining elements
    result.extend(left[i:])
    result.extend(right[j:])
    return result

print(merge_sort([38, 27, 43, 3, 9, 82, 10]))
# [3, 9, 10, 27, 38, 43, 82]
Quick Sort — O(n log n) average PYTHON
python3
def quick_sort(arr):
    """
    Quick sort: pick a pivot, partition around it, recursively sort.
    Time: O(n log n) average, O(n²) worst | Space: O(log n)
    NOT stable. In-place is more complex; this version uses lists.
    """
    if len(arr) <= 1:
        return arr
    
    pivot = arr[len(arr) // 2]     # choose middle element as pivot
    left  = [x for x in arr if x < pivot]   # elements < pivot
    mid   = [x for x in arr if x == pivot]  # elements == pivot
    right = [x for x in arr if x > pivot]   # elements > pivot
    
    return quick_sort(left) + mid + quick_sort(right)

print(quick_sort([3, 6, 8, 10, 1, 2, 1]))
# [1, 1, 2, 3, 6, 8, 10]

Ch 8: Binary Search
Find anything in a sorted collection in O(log n) time. One of the most elegant algorithms in CS.
⏱ 40 minutes

🔍 The Core Idea

On a sorted array, instead of checking every element (O(n)), you check the middle. If the target is smaller, search the left half. If larger, search the right half. Each step eliminates half the remaining elements.

With 1 billion elements, binary search finds any element in at most 30 steps. Linear search would take 1 billion steps.

python3 — binary search (iterative)
def binary_search(arr, target):
    """
    Returns index of target in sorted arr, or -1 if not found.
    REQUIRES: arr must be sorted!
    Time: O(log n) | Space: O(1)
    """
    lo, hi = 0, len(arr) - 1
    
    while lo <= hi:
        mid = (lo + hi) // 2    # avoid overflow (Python ints don't overflow, but good habit)
        
        if arr[mid] == target:
            return mid           # FOUND
        elif arr[mid] < target:
            lo = mid + 1         # search RIGHT half
        else:
            hi = mid - 1         # search LEFT half
    
    return -1   # not found

arr = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]
print(binary_search(arr, 23))   # 5 (index)
print(binary_search(arr, 99))   # -1

# Problem: You have heights [4,7,2,11,8] and n=3 piles.
# Find the minimum "capacity" K such that you can split
# the array into n groups each with sum ≤ K.
# Answer space: K ranges from max(heights) to sum(heights)

def can_split(arr, n, capacity):
    """Can we split arr into n groups each summing ≤ capacity?"""
    groups = 1
    current_sum = 0
    for x in arr:
        if x > capacity: return False   # single element too big
        if current_sum + x > capacity:
            groups += 1                  # start new group
            current_sum = x
        else:
            current_sum += x
    return groups <= n

def min_capacity(arr, n):
    lo, hi = max(arr), sum(arr)
    result = hi
    while lo <= hi:
        mid = (lo + hi) // 2
        if can_split(arr, n, mid):
            result = mid       # valid solution, try smaller
            hi = mid - 1
        else:
            lo = mid + 1       # not valid, need bigger capacity
    return result

print(min_capacity([4,7,2,11,8], 3))  # 11

def two_sum_sorted(arr, target):
    """Find two numbers that sum to target (arr is sorted)"""
    left, right = 0, len(arr) - 1
    
    while left < right:
        s = arr[left] + arr[right]
        if s == target:
            return [arr[left], arr[right]]
        elif s < target:
            left += 1    # need larger sum → move left pointer right
        else:
            right -= 1   # need smaller sum → move right pointer left
    return []

print(two_sum_sorted([1,2,3,4,6], 6))  # [2, 4]

def remove_duplicates_in_place(arr):
    """Remove duplicates from sorted array. Return new length."""
    if not arr: return 0
    slow = 0    # slow pointer: position of last unique element
    
    for fast in range(1, len(arr)):
        if arr[fast] != arr[slow]:
            slow += 1
            arr[slow] = arr[fast]
    
    return slow + 1

def max_water(heights):
    """Classic problem: find max area between any two bars"""
    left, right = 0, len(heights) - 1
    max_area = 0
    
    while left < right:
        h = min(heights[left], heights[right])
        w = right - left
        max_area = max(max_area, h * w)
        
        # Move the SHORTER bar inward (trying to find taller one)
        if heights[left] < heights[right]:
            left += 1
        else:
            right -= 1
    
    return max_area

# ─── FIXED WINDOW: window size is fixed ──────────────────
def max_sum_subarray(arr, k):
    """Max sum of any k consecutive elements. O(n)"""
    # Build first window
    window_sum = sum(arr[:k])
    max_sum = window_sum
    
    # Slide the window: add right element, remove left element
    for i in range(k, len(arr)):
        window_sum += arr[i]        # add new right element
        window_sum -= arr[i - k]    # remove old left element
        max_sum = max(max_sum, window_sum)
    
    return max_sum

print(max_sum_subarray([2,1,5,1,3,2], k=3))   # 9 (5+1+3)

# ─── VARIABLE WINDOW: expand/shrink based on condition ───
def longest_substring_no_repeat(s):
    """Longest substring without repeating characters. O(n)"""
    char_set = set()
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        # Shrink window while duplicate exists
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        
        char_set.add(s[right])
        max_len = max(max_len, right - left + 1)
    
    return max_len

print(longest_substring_no_repeat("abcabcbb"))  # 3 ("abc")
print(longest_substring_no_repeat("pwwkew"))    # 3 ("wke")

# ─── MIN WINDOW SUBSTRING ─────────────────────────────────
def min_window_sum(arr, target_sum):
    """Minimum length subarray with sum ≥ target. O(n)"""
    left = 0
    current_sum = 0
    min_len = float('inf')
    
    for right in range(len(arr)):
        current_sum += arr[right]
        
        while current_sum >= target_sum:
            min_len = min(min_len, right - left + 1)
            current_sum -= arr[left]
            left += 1
    
    return min_len if min_len != float('inf') else 0

Ch 10: Dynamic Programming
The most powerful — and feared — technique in competitive programming. Break it into subproblems. Remember the answers.
⏱ 75 minutes — GO SLOW

🧠 The Core Idea of DP

Dynamic Programming solves a problem by breaking it into overlapping subproblems and storing the results so you never solve the same subproblem twice (memoization).

When to use DP? The problem has: (1) Optimal substructure — optimal solution is built from optimal sub-solutions. (2) Overlapping subproblems — same smaller problems are solved multiple times.

Classic DP problems: Fibonacci, Coin Change, Knapsack, Longest Common Subsequence, Longest Increasing Subsequence.

def fib_naive(n):
    if n <= 1: return n

def fib_bottom_up(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[0], dp[1] = 0, 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]

# Given coins [1,5,10,25] and amount 41,
# Find minimum number of coins to make that amount.
# 41 = 25+10+5+1 → 4 coins

def coin_change(coins, amount):
    # dp[i] = minimum coins needed to make amount i
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0   # Base case: 0 coins to make amount 0
    
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                # option: use this coin, then dp[i - coin] coins for the rest
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] != float('inf') else -1

def lcs(s1, s2):
    """
    Longest Common Subsequence.
    "ABCBDAB" and "BDCABA" → "BCBA" or "BDAB" = length 4
    
    dp[i][j] = LCS of s1[:i] and s2[:j]
    """
    m, n = len(s1), len(s2)
    # Create (m+1) x (n+1) table, initialized to 0
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1   # characters match: extend
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])   # take best of skipping
    
    return dp[m][n]

print(lcs("ABCBDAB", "BDCABA"))   # 4

# ─── LIS: Longest Increasing Subsequence ──────────────────
def lis(arr):
    """
    Length of longest increasing subsequence.
    [10,9,2,5,3,7,101,18] → [2,5,7,101] = length 4
    """
    if not arr: return 0
    n = len(arr)
    dp = [1] * n   # each element alone is a subsequence of length 1
    
    for i in range(1, n):
        for j in range(i):
            if arr[j] < arr[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)

print(lis([10, 9, 2, 5, 3, 7, 101, 18]))   # 4

function isAnagram(s1, s2) {
    if (s1.length !== s2.length) return false;
    
    const count = {};
    for (let c of s1) count[c] = (count[c] || 0) + 1;
    for (let c of s2) {
        if (!count[c]) return false;
        count[c]--;
    }
    return true;
}

console.log(isAnagram("listen", "silent"));  // true
console.log(isAnagram("hello", "world"));    // false

python3 — prefix sum
# ─── PREFIX SUM ──────────────────────────────────────────
# Problem: Given array, answer many queries: "sum from index i to j?"
# Naive: O(n) per query
# Prefix sum: O(n) preprocessing, O(1) per query!

arr = [3, 1, 4, 1, 5, 9, 2, 6]

# Build prefix sum array
# prefix[i] = sum of arr[0..i-1]
n = len(arr)
prefix = [0] * (n + 1)
for i in range(n):
    prefix[i+1] = prefix[i] + arr[i]

print(prefix)
# [0, 3, 4, 8, 9, 14, 23, 25, 31]

# Range sum query [l, r] (0-indexed, inclusive):
def range_sum(l, r):
    return prefix[r+1] - prefix[l]

print(range_sum(0, 2))   # 3+1+4 = 8
print(range_sum(2, 5))   # 4+1+5+9 = 19
print(range_sum(3, 7))   # 1+5+9+2+6 = 23

# ─── Subarray Sum Equals K ────────────────────────────────
# Count subarrays with sum exactly k
def count_subarrays_sum_k(arr, k):
    """Uses prefix sum + hash map. O(n)"""
    count = 0
    curr_sum = 0
    seen = {0: 1}   # prefix sum 0 seen once (empty subarray)
    
    for x in arr:
        curr_sum += x
        # If (curr_sum - k) was seen before, we have a valid subarray
        count += seen.get(curr_sum - k, 0)
        seen[curr_sum] = seen.get(curr_sum, 0) + 1
    
    return count

print(count_subarrays_sum_k([1,1,1], 2))    # 2
print(count_subarrays_sum_k([1,2,3], 3))    # 2

for price in prices:
    if price < min_price:
